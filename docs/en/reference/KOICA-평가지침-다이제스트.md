## ⚠️ Nature of These Guidelines (read first, without fail)

This document is a **procedural/practical manual** for KOICA evaluation, not **a definitional document for evaluation criteria.** The guidelines themselves state that they "presuppose" and "apply mutatis mutandis" the source documents (p.4).

**So what is ❌ NOT in these guidelines:**
- The **detailed definitions** of each criterion, such as Effectiveness
- The stages, definitions, and score ranges of the project-performance **rating grade scale** (highly satisfactory to unsatisfactory, etc.)
- The **pool of standard evaluation questions** per criterion (only 2, for Relevance and Efficiency, are given as examples, p.72)

**→ The source documents for the above (the system is completed once they are additionally secured):**
- "Project Final Evaluation Guidelines" (2017.06)
- "Development Cooperation Project Evaluation Guidelines" (2008.11)
- "Project Planning, Monitoring, and Evaluation Methodology" (2009.11)
- "Integrated Evaluation Guidelines for International Development Cooperation" · "Integrated Evaluation Manual" · "Evaluation Ethics Guidelines," "Detailed Implementation Guidelines for Project Evaluation"

> **Principle:** Do **not** arbitrarily make up grading scales or definitions not in these guidelines. Apply the system's "no grade without evidence" to the application of these guidelines as well.

## 1. Evaluation Criteria Framework (p.52-53, Detailed Implementation Guidelines for Project Evaluation, Appended Form 3)

KOICA uses **the OECD DAC 5 criteria + cross-cutting issues.** (As a 2017 document, it **does not include Coherence**, which was added in 2019.)

| # | Evaluation criterion | KOICA detailed evaluation items |
|---|---------|-------------------|
| 1 | **Relevance** | (1) Conformity with the SDGs and donor/recipient policies (2) Appropriateness of the project plan |
| 2 | **Efficiency** | (1) Efficiency of resource use (2) Technical efficiency |
| 3 | **Effectiveness** ★ | (1) Project effectiveness based on performance-indicator measurement results (2) Effectiveness of project management |
| 4 | **Impact** | (1) Likelihood of achieving the overall objective |
| 5 | **Sustainability** | (1) Potential for self-reliant development (2) Needs and constraints for self-reliance |
| 6 | **Cross-cutting issues** | (1) Gender-sensitive impact (2) Environmental impact |

Examples of standard evaluation questions (p.72, source OECD 2007 — only Relevance and Efficiency are given. The following is the project's own summary):
- **Relevance**: Asks whether the project's objectives, selection, planning, and implementation fit the policies and priorities of the partner country and Korea, and whether they well captured the beneficiaries' needs.
- **Efficiency**: Asks whether the outputs were efficient relative to the resources invested (time, budget, personnel).

## 2. Effectiveness — Core of the System (p.53)

**KOICA-style Effectiveness = a two-axis structure:**
1. **Effectiveness based on performance-indicator measurement results** — the primary basis for the determination is the **actual value against the target value of the PDM performance indicators**.
2. **Effectiveness of project management** — the effectiveness of the process of output delivery, operation, and management.

Practical principles for the determination:
- Performance indicators are measured as actual against PDM target, and **the data source must always be stated** (p.23).
- **Distinguish** output achievement from outcome achievement.
- Avoid weak grounds such as simple satisfaction ("Are you satisfied? Yes/No") (p.22).

> ⚠️ The **detailed definition and grade cutoffs for Effectiveness are not in these guidelines.** Until the source guidelines are secured, do not assert grades; leave them as provisional, qualitative descriptions.

## 3. Evaluation Conduct Principles (AI Evaluator Rules)

| Principle | Content | Basis |
|------|------|------|
| **State evidence/sources** | State the information source for every analysis | p.23, p.52 |
| **Unverified = INFO flag** | For information hard to confirm, do not assert; mark as `INFO` and verify by other means | p.21 |
| **Balance of strengths/weaknesses** | Analyze not only strengths but also weaknesses appropriately | p.60 |
| **State limitations** | Clearly describe the limitations/constraints of conducting the evaluation | p.52, p.60 |
| **Impartiality** | Include the positions of all major stakeholders without bias | p.60 |
| **Independence/neutrality** | Evaluation judgments are independent of external influence; neutral questions | p.21, p.25 |
| **Anonymization** | Refer to survey subjects by initials instead of real names (Officer A, Teacher B) | p.30 |
| **Confidentiality** | Do not disclose acquired information externally without prior approval | p.26, p.50 |
| **Clarity** | Answer the evaluation questions clearly and describe them logically | p.60 |

## 4. Evaluation-Report Quality-Inspection Checklist ★ (p.60) — 100 points maximum

> **The best asset in these guidelines.** The quality-assurance criteria (QA) of the "Detailed Implementation Guidelines for Project Evaluation." To be transplanted verbatim as the rubric of the future **evaluation-report quality-inspector agent**.

| Evaluation item (points) | Detailed check | Points |
|---|---|---|
| **1. Clarity of answers to evaluation criteria/principles (30)** | Did it provide a clear answer to each evaluation question | 10 |
| | Did it clearly explain the performance in light of the evaluation principles/criteria | 10 |
| | Is the narrative clear and logically coherent so the reader easily understands it | 10 |
| **2. Transparency of the evaluation survey method (15)** | Is the chosen evaluation methodology valid | 10 |
| | Did it provide grounds befitting the data and methodology | 5 |
| **3. Justification of the composition/conclusions (25)** | Is the composition logical and concise | 10 |
| | Did it support the conclusions with objective grounds | 10 |
| | Did it clearly disclose the limitations/constraints encountered while conducting the evaluation | 5 |
| **4. Impartiality of the content (15)** | Did it include the positions of all stakeholders without leaning to one side | 10 |
| | Did it point out not only strengths but also weaknesses | 5 |
| **5. Fidelity to the evaluation plan (15)** | Did it answer the evaluation questions in the plan with logical conclusions | 7 |
| | Was it written following the methodology in the plan | 8 |
| **Total** | | **100** |

- **Pass line: 80 points.** Below 80 → second review → still below on the second review → **not published** (p.41, p.60).
- Review-opinion format: per-item 〈review opinion〉 + 〈items for later supplementation〉, with the action classified as **reflected / not reflected / partially reflected** (p.61).

## 5. Evaluation Procedure (5 stages, p.5-6)

**① Evaluation planning → ② Evaluation design → ③ Evaluation conduct → ④ Results reporting → ⑤ Feedback**

| Stage | Core |
|------|------|
| ① Planning | Based on the annual plan, an individual evaluation plan (overview, scope, logic model, evaluation matrix, budget, schedule); RFP → bid → contract |
| ② Design | Finalize the evaluation matrix, methods, logic model; inception report and inception meeting |
| ③ Conduct | Domestic survey (literature, interviews, surveys → data analysis) → field survey (KOICA plays an observer role) |
| ④ Reporting | Interim report → draft evaluation report → **review (100 pts / 80 pts)** → supplementation → KOICA publishes in a batch |
| ⑤ Feedback | Recommendations in an operational form; designation and monitoring of feedback tasks |
